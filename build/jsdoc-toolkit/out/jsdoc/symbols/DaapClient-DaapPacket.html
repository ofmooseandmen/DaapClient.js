<inner> DaapClient-DaapPacket(chunk, start)
The DAAP response definition: a packet organized by 'codes' and 'values'. The basic format of DAAP is: 
  4 byte tag | size (4 byte int) | data
where the length of the data is determined by the size. The 4 byte tag can be represented as either an integer, or as a four character string.

The data can itself be either:

  • the actual transmitted data: an int or a String
  • a sequence of chunks (from 1 to n)
Parameters:
chunk
{String} a string representing a DAAP chunk (code + size + data).
start
{int} the offset at which to start reading the chunk - if omitted, 0 is assumed
Throws:
EndOfPacketException if the string does not hold enough information to extract code + data.
See:
DAAP Protocol documentation v0.2

readNextChunk()
Read the next chunk within this chunk.

Note: it is up to the user to use this method as long as the data associated to the returned chunk do contain yet other chunks. Once the return chunk contains the actual transmitted data use DaapPacket#convertToInt() or DaapPacket#convertToString(). Calling this method on an actual data will throw a EndOfPacketException.
